## 1. Обзор рынка

Рынок инструментов для управления знаниями (knowledge management) и документации разделён на несколько кластеров:

| Кластер | Продукты | Характеристика |
|---|---|---|
| Корпоративные вики | Confluence, GitBook | Enterprise-фокус, сложная настройка |
| Универсальные workspaces | Notion, Obsidian | Гибкость, заметки + задачи + базы |
| Open-source вики | MediaWiki | Бесплатно, но высокая цена владения |
| **AI-native** | **AI Portal** | **Генерация контента через промпт** |

## 3. УТП AI Portal — Уникальное торговое предложение

### Основное УТП
> **AI Portal — первый коробочный AI-портал, где страницы и пространства создаются через промпт.**

Вместо ручной структуры папок, пространств и страниц — пользователь описывает желаемый контент естественным языком, и AI генерирует структуру и наполнение автоматически.

### Ключевые отличия от конкурентов

| Аспект | Конкуренты | AI Portal |
|---|---|---|
| **Создание контента** | Ручное, шаблонное | AI-генерация из промпта |
| **Структура** | Жёсткая иерархия | Динамическая, адаптивная |
| **Время до первой страницы** | Минуты/часы | Секунды |
| **Обучение пользователей** | Требуется (структура, макросы) | Не требуется (естественный язык) |
| **Поддержка знаний** | Пользователь должен знать где искать | AI помогает найти и сгенерировать |
| **Развёртывание** | Сложная конфигурация | Коробочное решение |

### Дополнительные преимущества
- **Self-hosted по умолчанию** — данные остаются у клиента
- **Коробочное решение** — не нужно集成ровать 5-10 разных инструментов
- **Фокус на B2B** — не просто заметки, а документация для бизнеса

## 4. Целевая аудитория и Personas

### Сегменты рынка

#### 4.1. Малый и средний бизнес (SMB, 10-500 сотрудников)
- **Боли:** Нет ресурсов на сложную IT-инфраструктуру, нужен простой и быстрый способ документировать процессы
- **Потребность:** «Хочу задокументировать процесс онбординга, но не хочу тратить часы на оформление»

#### 4.2. IT-компании и стартапы (50-200 сотрудников)
- **Боли:** Confluence слишком тяжёлый, Notion не подходит для enterprise, GitBook сложен для нетехнических команд
- **Потребность:** Единое место для технической и бизнес-документации

#### 4.3. Отделы поддержки и customer success
- **Боли:** База знаний устарела, сотрудники не хотят её поддерживать, поиск не работает
- **Потребность:** Быстро создавать и обновлять FAQ по продукту

#### 4.4. HR и внутренние коммуникации
- **Боли:** Онбординг новых сотрудников занимает долго, информация разбросана
- **Потребность:** Создать раздел «Наша команда» или «Процессы компании» за минуты

  - «Сгенерируй страницу с процессом code review»
  - «Найди документацию по нашему API авторизации»
- **Критерии выбора:** Скорость развёртывания, AI-генерация, простота поиска

## 5. Выводы

### Рыночная возможность
AI Portal занимает нишу между тяжёлыми enterprise-решениями (Confluence) и простыми инструментами для личного использования (Obsidian). УТП — AI-генерация контента из промпта — не имеет прямых аналогов среди коробочных self-hosted решений.

### Конкурентные преимущества
1. **AI-first подход** — не дополнение, а ядро продукта
2. **Коробочное self-hosted решение** — данные у клиента, без облачных зависимостей
3. **Низкий порог входа** — не нужно обучать сотрудников структуре и макросам
4. **B2B-фокус** — не просто заметки, а инструмент для бизнес-документации

### Риски
- Notion и Confluence могут добавить AI-генерацию (Confluence уже имеет AI Assistant)
- Рынок фрагментирован — нужно чётко позиционироваться
- Self-hosted требует поддержки и обновлений

### Рекомендации
1. Фокус на B2B сегмент (50-500 сотрудников)
2. Акцент на self-hosted + AI-first в маркетинге
3. Быстрый time-to-value — демонстрация генерации страницы за секунды
4. Интеграции с популярными корпоративными инструментами (Slack, Jira, GitHub)
